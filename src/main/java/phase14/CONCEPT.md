## 🐳 Docker 核心概念

### 容器 vs 虚拟机

```
┌─────────────────────────────────────────────────────────────────────┐
│                                                                     │
│         虚拟机                           容器                       │
│  ┌─────────────────────┐        ┌─────────────────────┐            │
│  │   App A  │  App B   │        │   App A  │  App B   │            │
│  ├─────────────────────┤        ├─────────────────────┤            │
│  │ Guest OS │ Guest OS │        │       运行时         │            │
│  ├─────────────────────┤        ├─────────────────────┤            │
│  │      Hypervisor     │        │      Docker         │            │
│  ├─────────────────────┤        ├─────────────────────┤            │
│  │      Host OS        │        │      Host OS        │            │
│  ├─────────────────────┤        ├─────────────────────┤            │
│  │      硬件            │        │      硬件            │            │
│  └─────────────────────┘        └─────────────────────┘            │
│                                                                     │
│  ❌ 资源开销大 (GB级)            ✅ 轻量级 (MB级)                   │
│  ❌ 启动慢 (分钟级)              ✅ 启动快 (秒级)                   │
│  ✅ 完全隔离                     ⚠️ 共享内核                       │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### Docker 核心组件

| 组件                 | 说明                             |
| -------------------- | -------------------------------- |
| **镜像 (Image)**     | 只读模板，包含运行应用所需的一切 |
| **容器 (Container)** | 镜像的运行实例                   |
| **Dockerfile**       | 构建镜像的脚本                   |
| **仓库 (Registry)**  | 存储和分发镜像                   |
| **Docker Compose**   | 定义和运行多容器应用             |

### Docker 常用命令

```bash
# 镜像操作
docker pull nginx:latest      # 拉取镜像
docker images                 # 查看本地镜像
docker rmi nginx:latest       # 删除镜像
docker build -t myapp:1.0 .   # 构建镜像

# 容器操作
docker run -d -p 8080:80 nginx    # 后台运行容器
docker ps                         # 查看运行中的容器
docker ps -a                      # 查看所有容器
docker stop <container_id>        # 停止容器
docker rm <container_id>          # 删除容器
docker logs -f <container_id>     # 查看日志
docker exec -it <id> /bin/bash    # 进入容器

# 数据卷
docker volume create mydata       # 创建数据卷
docker run -v mydata:/data nginx  # 挂载数据卷
```

## ☸️ Kubernetes 核心概念

### K8s 架构

```
┌─────────────────────────────────────────────────────────────────────┐
│                        Kubernetes 集群                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  Control Plane (Master)                                             │
│  ┌─────────────┬─────────────┬─────────────┬─────────────┐         │
│  │ API Server  │ Scheduler   │ Controller  │   etcd      │         │
│  │             │             │  Manager    │  (存储)     │         │
│  └─────────────┴─────────────┴─────────────┴─────────────┘         │
│                              │                                      │
│                              ↓                                      │
│  Worker Nodes                                                       │
│  ┌─────────────────────────────────────────────────────────┐       │
│  │ Node 1                    │ Node 2                      │       │
│  │ ┌─────────────────────┐   │ ┌─────────────────────┐     │       │
│  │ │ kubelet  │ kube-proxy│  │ │ kubelet  │ kube-proxy│    │       │
│  │ ├─────────────────────┤   │ ├─────────────────────┤     │       │
│  │ │ Pod  Pod  Pod       │   │ │ Pod  Pod  Pod       │     │       │
│  │ └─────────────────────┘   │ └─────────────────────┘     │       │
│  └─────────────────────────────────────────────────────────┘       │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### K8s 核心资源

| 资源            | 说明               | 用途                 |
| --------------- | ------------------ | -------------------- |
| **Pod**         | 最小部署单元       | 一个或多个容器的集合 |
| **Deployment**  | 无状态应用管理     | 声明式更新、回滚     |
| **Service**     | 服务发现和负载均衡 | 暴露 Pod 为服务      |
| **ConfigMap**   | 配置管理           | 存储非敏感配置       |
| **Secret**      | 敏感信息管理       | 存储密码、密钥       |
| **Ingress**     | 入口控制           | HTTP/HTTPS 路由      |
| **PV/PVC**      | 持久化存储         | 数据持久化           |
| **StatefulSet** | 有状态应用管理     | 数据库、消息队列     |

### K8s 常用命令

```bash
# 集群信息
kubectl cluster-info              # 集群信息
kubectl get nodes                 # 查看节点

# 资源操作
kubectl apply -f deployment.yaml  # 应用配置
kubectl get pods                  # 查看 Pod
kubectl get svc                   # 查看 Service
kubectl get deployments           # 查看 Deployment

# 调试
kubectl describe pod <name>       # Pod 详情
kubectl logs <pod-name>           # 查看日志
kubectl exec -it <pod> -- bash    # 进入容器

# 扩缩容
kubectl scale deployment myapp --replicas=5

# 更新
kubectl set image deployment/myapp myapp=myapp:v2
kubectl rollout status deployment/myapp
kubectl rollout undo deployment/myapp
```

## 🔄 CI/CD 概念

### CI/CD 流程

```
┌─────────────────────────────────────────────────────────────────────┐
│                        CI/CD Pipeline                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   代码提交     构建      测试      镜像      部署      监控         │
│      │         │         │         │         │         │           │
│      ↓         ↓         ↓         ↓         ↓         ↓           │
│   ┌─────┐   ┌─────┐   ┌─────┐   ┌─────┐   ┌─────┐   ┌─────┐       │
│   │ Git │ → │Build│ → │Test │ → │Push │ → │Deploy│→ │Monitor│      │
│   └─────┘   └─────┘   └─────┘   └─────┘   └─────┘   └─────┘       │
│                                                                     │
│   ←────── CI (持续集成) ──────→ ←───── CD (持续部署) ─────→         │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### CI/CD 工具

| 工具               | 类型         | 特点               |
| ------------------ | ------------ | ------------------ |
| **Jenkins**        | 老牌 CI/CD   | 插件丰富，自托管   |
| **GitLab CI**      | 集成 CI/CD   | 与 GitLab 深度集成 |
| **GitHub Actions** | 云端 CI/CD   | 与 GitHub 深度集成 |
| **ArgoCD**         | GitOps       | K8s 原生 CD        |
| **Tekton**         | 云原生 CI/CD | K8s 原生           |

## 🌐 服务网格 (Service Mesh)

### 为什么需要服务网格？

```
微服务架构的挑战:
• 服务发现
• 负载均衡
• 熔断降级
• 链路追踪
• 安全通信 (mTLS)
• 流量管理

服务网格通过 Sidecar 代理统一解决这些问题
```

### Istio 架构

```
┌─────────────────────────────────────────────────────────────────────┐
│                           Istio                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  Control Plane                                                      │
│  ┌──────────────────────────────────────────────────────────────┐  │
│  │                         istiod                                │  │
│  │  (Pilot + Citadel + Galley)                                  │  │
│  └──────────────────────────────────────────────────────────────┘  │
│                              ↑↓                                     │
│  Data Plane                                                         │
│  ┌─────────────────────┐    ┌─────────────────────┐                │
│  │  ┌───────┬───────┐  │    │  ┌───────┬───────┐  │                │
│  │  │Service│ Envoy │← ──── →│ Envoy  │Service│  │                │
│  │  │  A    │(Sidecar)│     │(Sidecar)│  B    │  │                │
│  │  └───────┴───────┘  │    │  └───────┴───────┘  │                │
│  │       Pod 1         │    │       Pod 2         │                │
│  └─────────────────────┘    └─────────────────────┘                │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 📖 学习路径

```
1️⃣ Docker 基础
   └── 安装 Docker → 编写 Dockerfile → 构建镜像 → Docker Compose

2️⃣ Kubernetes 入门
   └── 安装 Minikube → 部署第一个应用 → Service 暴露

3️⃣ K8s 进阶
   └── ConfigMap/Secret → 持久化存储 → Ingress

4️⃣ CI/CD 实践
   └── GitHub Actions → 自动构建 → 自动部署到 K8s

5️⃣ 服务网格 (可选)
   └── 安装 Istio → 流量管理 → 链路追踪
```
