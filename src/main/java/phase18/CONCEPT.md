## 🔗 分布式链路追踪

### OpenTelemetry 概念

```
┌─────────────────────────────────────────────────────────────┐
│                      分布式链路追踪                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   请求进入                                                   │
│      │                                                      │
│      ▼                                                      │
│   ┌──────────────────────────────────────────────────┐     │
│   │ Trace (一次完整请求)    TraceId: abc123           │     │
│   │ ├── Span A: API Gateway        [0ms - 500ms]     │     │
│   │ │   ├── Span B: User Service   [50ms - 200ms]    │     │
│   │ │   │   └── Span C: MySQL      [80ms - 150ms]    │     │
│   │ │   └── Span D: Order Service  [220ms - 450ms]   │     │
│   │ │       └── Span E: Redis      [250ms - 280ms]   │     │
│   │ └── Total: 500ms                                 │     │
│   └──────────────────────────────────────────────────┘     │
│                                                             │
└─────────────────────────────────────────────────────────────┘

核心概念:
- Trace: 一次请求的完整路径
- Span: Trace 中的一个操作单元
- Context: 在服务间传递的追踪上下文 (TraceId + SpanId)
```

### 上下文传播

```java
// HTTP Header 传播追踪上下文
Request                     Response
┌─────────────────┐        ┌─────────────────┐
│ traceparent:    │   →    │ Service B       │
│ 00-abc123-def-01│        │ 解析 Header     │
│                 │        │ 创建子 Span     │
└─────────────────┘        └─────────────────┘

W3C Trace Context 格式:
traceparent: {version}-{trace-id}-{parent-span-id}-{flags}
```

## 📈 Prometheus 指标监控

### 指标类型

```
┌─────────────────────────────────────────────────────────────┐
│                    Prometheus 指标类型                       │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   Counter (计数器)                                          │
│   ┌──────────────────────────────────────────┐             │
│   │  只增不减，如请求总数                       │             │
│   │  http_requests_total{method="GET"} = 1000  │             │
│   └──────────────────────────────────────────┘             │
│                                                             │
│   Gauge (仪表盘)                                            │
│   ┌──────────────────────────────────────────┐             │
│   │  可增可减，如当前连接数                     │             │
│   │  active_connections = 42                   │             │
│   └──────────────────────────────────────────┘             │
│                                                             │
│   Histogram (直方图)                                        │
│   ┌──────────────────────────────────────────┐             │
│   │  统计分布，如响应时间百分位                  │             │
│   │  http_request_duration_bucket{le="0.1"}    │             │
│   │  http_request_duration_bucket{le="0.5"}    │             │
│   │  http_request_duration_bucket{le="1.0"}    │             │
│   └──────────────────────────────────────────┘             │
│                                                             │
│   Summary (摘要)                                            │
│   ┌──────────────────────────────────────────┐             │
│   │  预计算百分位，如 P99 延迟                  │             │
│   │  rpc_duration_seconds{quantile="0.99"}     │             │
│   └──────────────────────────────────────────┘             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### PromQL 常用查询

```promql
# 1. 请求速率 (QPS)
rate(http_requests_total[5m])

# 2. 错误率
rate(http_requests_total{status="500"}[5m]) / rate(http_requests_total[5m])

# 3. P99 延迟
histogram_quantile(0.99, rate(http_request_duration_bucket[5m]))

# 4. 内存使用率
(node_memory_total - node_memory_available) / node_memory_total * 100
```

## 📝 结构化日志

### 传统日志 vs 结构化日志

```
传统日志:
2024-01-15 10:23:45 INFO User 12345 ordered product abc for $99.99

结构化日志 (JSON):
{
  "timestamp": "2024-01-15T10:23:45.123Z",
  "level": "INFO",
  "message": "User ordered product",
  "userId": 12345,
  "productId": "abc",
  "amount": 99.99,
  "traceId": "abc123",
  "spanId": "def456"
}

优势:
- 机器可解析，支持高效查询
- 便于聚合分析
- 与链路追踪关联
```

## 🚨 告警体系设计

### 告警分级

| 级别    | 响应时间 | 通知方式        | 示例           |
| ------- | -------- | --------------- | -------------- |
| P0 紧急 | 5 分钟   | 电话 + 短信     | 全站不可用     |
| P1 严重 | 15 分钟  | 短信 + 即时通讯 | 核心功能故障   |
| P2 一般 | 1 小时   | 即时通讯        | 非核心功能异常 |
| P3 提醒 | 工作时间 | 邮件            | 性能下降/预警  |

### 告警规则示例

```yaml
# Prometheus Alertmanager 配置
groups:
  - name: api-alerts
    rules:
      # 错误率超过 5%
      - alert: HighErrorRate
        expr: rate(http_requests_total{status="500"}[5m]) / rate(http_requests_total[5m]) > 0.05
        for: 5m
        labels:
          severity: critical
        annotations:
          summary: "API 错误率过高"

      # P99 延迟超过 1 秒
      - alert: HighLatency
        expr: histogram_quantile(0.99, rate(http_request_duration_bucket[5m])) > 1
        for: 10m
        labels:
          severity: warning
```
