## 🌊 流批一体

```
Flink vs Spark Streaming
═════════════════════════════════════════════════════════

Spark Streaming: 微批处理
┌─────┐┌─────┐┌─────┐┌─────┐
│Batch││Batch││Batch││Batch│  → 按时间切分批次
└─────┘└─────┘└─────┘└─────┘
延迟：秒级

Flink: 真正流处理
════════════════════════════════════►
每条数据到达立即处理
延迟：毫秒级
```

## ⏰ 时间语义

| 时间类型            | 说明            | 使用场景     |
| ------------------- | --------------- | ------------ |
| **Event Time**      | 事件发生时间    | 有序性要求高 |
| **Processing Time** | 处理时间        | 简单场景     |
| **Ingestion Time**  | 进入 Flink 时间 | 折中方案     |

### Watermark 水印

```
处理乱序数据的机制

事件流: [1,2,5,3,4,7,6,8...]
         ^
         乱序事件

Watermark(t) = 当前最大事件时间 - 允许延迟
表示时间 t 之前的数据已经全部到达
```

## 🪟 窗口类型

```
滚动窗口 (Tumbling)
═════════════════════════════════════════════════════════
|  Window 1  |  Window 2  |  Window 3  |
不重叠，连续

滑动窗口 (Sliding)
═════════════════════════════════════════════════════════
|  Window 1        |
      |  Window 2        |
            |  Window 3        |
可重叠

会话窗口 (Session)
═════════════════════════════════════════════════════════
|  Session 1  |    Gap    |  Session 2  |
按活动间隙划分
```

## 💾 状态管理

### 状态类型

| 类型              | 说明     | 示例     |
| ----------------- | -------- | -------- |
| **ValueState**    | 单值状态 | 计数器   |
| **ListState**     | 列表状态 | 事件缓存 |
| **MapState**      | Map 状态 | 用户会话 |
| **ReducingState** | 聚合状态 | 求和     |

### 状态后端

| 后端                            | 存储位置   | 适用场景         |
| ------------------------------- | ---------- | ---------------- |
| **HashMapStateBackend**         | JVM 堆内存 | 小状态，快速访问 |
| **EmbeddedRocksDBStateBackend** | 本地磁盘   | 大状态，可扩展   |

## 📝 Flink vs Spark

| 特性             | Flink      | Spark Streaming |
| ---------------- | ---------- | --------------- |
| **处理模型**     | 真正流处理 | 微批处理        |
| **延迟**         | 毫秒级     | 秒级            |
| **Exactly-Once** | 原生支持   | 需额外配置      |
| **状态管理**     | 强大内置   | 有限支持        |
| **API 丰富度**   | 流为主     | 批为主          |
| **生态成熟度**   | 快速发展   | 更成熟          |
