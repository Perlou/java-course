## 🎯 MySQL 索引体系

```
┌─────────────────────────────────────────────────────────────────────┐
│                        B+ 树索引结构                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│                      ┌─────────────┐                                │
│                      │ 25 | 50 | 75│  ← 根节点 (非叶子)             │
│                      └──┬──┬──┬──┬─┘                                │
│                ┌────────┘  │  │  └────────┐                         │
│                ↓           ↓  ↓           ↓                         │
│         ┌──────────┐  ┌──────────┐  ┌──────────┐                    │
│         │ 10|15|20 │  │ 30|35|40 │  │ 80|85|90 │  ← 非叶子节点      │
│         └────┬─────┘  └────┬─────┘  └────┬─────┘                    │
│              ↓             ↓             ↓                          │
│         ┌──────────┐  ┌──────────┐  ┌──────────┐                    │
│         │数据|数据 │→ │数据|数据 │→ │数据|数据 │  ← 叶子节点(链表)  │
│         └──────────┘  └──────────┘  └──────────┘                    │
│                                                                     │
│  特点:                                                              │
│  • 叶子节点存储实际数据，形成有序链表                               │
│  • 非叶子节点只存储索引，不存储数据                                 │
│  • 查询时间复杂度 O(log n)                                          │
│  • 范围查询效率高（链表遍历）                                       │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 索引类型

| 索引类型     | 描述                     | 示例           |
| ------------ | ------------------------ | -------------- |
| **主键索引** | 聚簇索引，数据按主键排序 | `PRIMARY KEY`  |
| **唯一索引** | 不允许重复值             | `UNIQUE INDEX` |
| **普通索引** | 最基本的索引             | `INDEX`        |
| **联合索引** | 多列组合索引             | `(a, b, c)`    |
| **覆盖索引** | 索引包含查询所需全部字段 | 避免回表       |

### 最左前缀原则

```sql
-- 联合索引 (a, b, c)
SELECT * FROM t WHERE a = 1                 -- ✅ 使用索引
SELECT * FROM t WHERE a = 1 AND b = 2       -- ✅ 使用索引
SELECT * FROM t WHERE a = 1 AND b = 2 AND c = 3  -- ✅ 使用索引
SELECT * FROM t WHERE b = 2                 -- ❌ 无法使用
SELECT * FROM t WHERE a = 1 AND c = 3       -- ⚠️ 只能用到 a
```

### MVCC 原理

```
┌─────────────────────────────────────────────────────────────────────┐
│                      MVCC 多版本并发控制                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  每行记录包含隐藏列:                                                │
│  • DB_TRX_ID: 最后修改该行的事务ID                                  │
│  • DB_ROLL_PTR: 指向 undo log 的指针                                │
│                                                                     │
│  版本链:                                                            │
│  ┌─────────┐      ┌─────────┐      ┌─────────┐                      │
│  │ 当前版本 │ ───→ │ 上一版本 │ ───→ │ 更早版本 │                      │
│  │ TRX=100 │      │ TRX=90  │      │ TRX=80  │                      │
│  └─────────┘      └─────────┘      └─────────┘                      │
│                                                                     │
│  Read View:                                                         │
│  • 事务开始时创建快照                                               │
│  • 根据快照判断数据版本可见性                                       │
│  • 实现非锁定一致性读                                               │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 动态 SQL

```xml
<!-- if 条件 -->
<select id="findUsers" resultType="User">
    SELECT * FROM users
    <where>
        <if test="name != null">AND name = #{name}</if>
        <if test="age != null">AND age = #{age}</if>
    </where>
</select>

<!-- foreach 批量 -->
<insert id="batchInsert">
    INSERT INTO users (name, age) VALUES
    <foreach collection="list" item="user" separator=",">
        (#{user.name}, #{user.age})
    </foreach>
</insert>
```

## ⚡ MyBatis-Plus

```java
// 继承 BaseMapper 即可获得 CRUD
public interface UserMapper extends BaseMapper<User> {
    // 无需编写任何方法
}

// 条件构造器
QueryWrapper<User> wrapper = new QueryWrapper<>();
wrapper.eq("status", 1)
       .like("name", "张")
       .orderByDesc("create_time");
List<User> users = userMapper.selectList(wrapper);

// Lambda 写法 (推荐)
LambdaQueryWrapper<User> lambda = Wrappers.lambdaQuery();
lambda.eq(User::getStatus, 1)
      .like(User::getName, "张");
```

## 📊 SQL 优化要点

```
┌─────────────────────────────────────────────────────────────────────┐
│                      EXPLAIN 关键字段                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  type (访问类型，性能从好到差):                                      │
│  ────────────────────────────────────────────────────               │
│  system > const > eq_ref > ref > range > index > ALL               │
│                                                                     │
│  key: 实际使用的索引                                                │
│  rows: 预估扫描行数                                                 │
│  Extra:                                                             │
│    • Using index      → 覆盖索引 ✅                                 │
│    • Using where      → WHERE 过滤                                  │
│    • Using filesort   → 需要额外排序 ❌                             │
│    • Using temporary  → 使用临时表 ❌                               │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘

优化原则:
• 避免 SELECT *
• 避免在 WHERE 子句中对字段进行函数操作
• 避免使用 != 或 <> (可能导致不走索引)
• 使用 LIMIT 限制结果集
• 批量操作代替循环单条操作
```

## 📝 核心要点

1. **索引优化**: 理解 B+ 树，善用联合索引和覆盖索引
2. **事务隔离**: RR 是 MySQL 默认级别，MVCC 实现无锁读
3. **MyBatis**: 半自动 ORM，动态 SQL 灵活控制
4. **MyBatis-Plus**: 快速开发，减少样板代码
5. **SQL 优化**: 看懂 EXPLAIN，避免全表扫描
6. **分库分表**: 数据量大时的水平扩展方案
