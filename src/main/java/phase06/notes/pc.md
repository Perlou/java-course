# 程序计数器（Program Counter，PC）

## 一、基本定义

程序计数器是 **CPU 中的一个特殊寄存器**，用于存储 **下一条将要执行的指令的内存地址**。

```
┌─────────────────────────────────────────────┐
│                    CPU                       │
│  ┌─────────────────────────────────────┐    │
│  │  程序计数器 (PC)                     │    │
│  │  ┌─────────────────────────────┐    │    │
│  │  │   0x0040102A (下一条指令地址) │    │    │
│  │  └─────────────────────────────┘    │    │
│  └─────────────────────────────────────┘    │
└─────────────────────────────────────────────┘
```

---

## 二、主要作用

| 作用         | 说明                                |
| ------------ | ----------------------------------- |
| **指令定位** | 告诉 CPU 下一条指令在内存的哪个位置 |
| **顺序执行** | 自动递增，保证程序按顺序执行        |
| **流程控制** | 支持跳转、循环、函数调用等操作      |
| **线程恢复** | 线程切换后能恢复到正确的执行位置    |

---

## 三、工作原理

```
内存中的指令序列：
┌──────────┬─────────────────────┐
│  地址    │       指令          │
├──────────┼─────────────────────┤
│  1000    │  MOV A, 10         │  ← PC=1000，执行这条
│  1001    │  ADD A, 5          │  ← PC=1001，接着执行
│  1002    │  JMP 2000          │  ← PC=1002，跳转指令
│  1003    │  ...               │
│  ...     │  ...               │
│  2000    │  SUB A, 3          │  ← PC=2000，跳转到这里
└──────────┴─────────────────────┘
```

### 执行流程：

```
1. 读取 PC 指向的指令
2. PC 自动 +1（或根据指令长度递增）
3. 执行当前指令
4. 如果是跳转指令，修改 PC 为目标地址
5. 重复步骤 1
```

---

## 四、在 JVM 中的程序计数器

在 Java 虚拟机中，PC 寄存器有特殊含义：

```
┌─────────────────────────────────────┐
│           JVM 运行时数据区            │
├─────────────────────────────────────┤
│  ┌─────────────┐  ← 线程私有         │
│  │ PC 寄存器   │  存储字节码指令地址   │
│  └─────────────┘                     │
│  每个线程都有独立的 PC                │
└─────────────────────────────────────┘
```

**特点：**

- ✅ 线程私有（每个线程独立拥有）
- ✅ 执行 Java 方法时：记录字节码指令地址
- ✅ 执行 Native 方法时：值为 **undefined**
- ✅ **唯一不会发生 OOM 的区域**

## 五、为什么需要程序计数器？

```
场景：多线程执行

线程A                        线程B
┌─────┐                     ┌─────┐
│PC=50│ ──暂停──┐     ┌──── │PC=80│
└─────┘         │     │     └─────┘
                ↓     ↓
           ┌──────────────┐
           │     CPU      │
           │  时间片轮转   │
           └──────────────┘
                ↓
        线程A恢复时，PC=50
        知道从哪里继续执行！
```

---

## 总结

| 特性         | 说明                 |
| ------------ | -------------------- |
| **本质**     | CPU 中的寄存器       |
| **存储内容** | 下一条指令的地址     |
| **更新方式** | 自动递增 或 跳转修改 |
| **线程安全** | 每个线程独立拥有     |

程序计数器是程序能够 **有序执行** 和 **正确恢复** 的关键！
