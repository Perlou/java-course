# JVM 方法区详解

## 一、基本概念

方法区（Method Area）是 JVM 规范中定义的一块内存区域，用于存储已被加载的**类信息、常量、静态变量、即时编译器编译后的代码**等数据。

```
│                         JVM 内存结构                              │
├─────────────────────────────────────────────────────────────────┤
│  ┌──────────────────────────────────────────────────────────┐   │
│  │                    线程共享区域                            │   │
│  │  ┌─────────────────────┐  ┌───────────────────────────┐  │   │
│  │  │        堆           │  │        方法区              │  │   │

│  │  │                    │  │  ・静态变量                 │  │   │
```

## 三、方法区的演变（重点）

### 版本对比

| 版本       | 实现方式            | 存储位置 | 特点                   |
| ---------- | ------------------- | -------- | ---------------------- |
| **JDK 6**  | 永久代（PermGen）   | JVM 内存 | 与堆连续，大小固定     |
| **JDK 7**  | 永久代（过渡期）    | JVM 内存 | 字符串常量池移至堆     |
| **JDK 8+** | 元空间（Metaspace） | 本地内存 | 使用系统内存，动态扩展 |

## 四、运行时常量池详解

```java
public class ConstantPoolDemo {
    // 字面量
    private String name = "Hello";          // 字符串常量
    private final int MAX_SIZE = 100;       // final常量

    // 符号引用
    private Object obj;                     // 类引用
    private void test() {}                  // 方法引用
}
```

## 五、字符串常量池（String Pool）

```java
public class StringPoolDemo {
    public static void main(String[] args) {
        // JDK 6: 永久代
        // JDK 7+: 堆

        String s1 = "abc";              // 常量池
        String s2 = "abc";              // 常量池（同一引用）
        String s3 = new String("abc");  // 堆中新对象
        String s4 = s3.intern();        // 返回常量池引用

        System.out.println(s1 == s2);   // true
        System.out.println(s1 == s3);   // false
        System.out.println(s1 == s4);   // true
    }
}
```

### 内存布局

```
┌─────────────────────────────────────────────────────────────┐
│                          堆 (Heap)                           │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│   ┌─────────────────────────────┐                           │
│   │     字符串常量池             │                           │
│   │   ┌───────────────────┐     │                           │
│   │   │  "abc" ←──────────│─────│── s1, s2, s4 指向这里     │
│   │   └───────────────────┘     │                           │
│   └─────────────────────────────┘                           │
│                                                              │
│   ┌───────────────────┐                                     │
│   │ String对象 "abc"  │ ←───────── s3 指向这里              │
│   │  (new创建)        │                                     │
│   └───────────────────┘                                     │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## 六、相关 JVM 参数

### JDK 7 及之前（永久代）

```bash
# 设置永久代初始大小
-XX:PermSize=64m

# 设置永久代最大大小
-XX:MaxPermSize=256m
```

### JDK 8+（元空间）

```bash
# 设置元空间初始大小
-XX:MetaspaceSize=128m

# 设置元空间最大大小（默认无限制）
-XX:MaxMetaspaceSize=256m

# 设置元空间触发GC的阈值
-XX:MinMetaspaceFreeRatio=40
-XX:MaxMetaspaceFreeRatio=70
```

## 七、方法区的垃圾回收

### 回收条件（类卸载条件）

```
类卸载必须同时满足以下三个条件:

┌─────────────────────────────────────────────────────────────┐
│  ✓ 1. 该类所有实例都已被回收（堆中不存在该类对象）            │
├─────────────────────────────────────────────────────────────┤
│  ✓ 2. 加载该类的ClassLoader已被回收                         │
├─────────────────────────────────────────────────────────────┤
│  ✓ 3. 该类的Class对象没有被任何地方引用                      │
│      （无法通过反射访问该类）                                │
└─────────────────────────────────────────────────────────────┘
```

## 八、常见 OOM 问题

### 1. 永久代溢出（JDK 7 及之前）

```java
// 不断创建新类导致
java.lang.OutOfMemoryError: PermGen space
```

### 2. 元空间溢出（JDK 8+）

```java
// 动态生成大量类（如CGLib）
java.lang.OutOfMemoryError: Metaspace
```

### 模拟代码

```java
public class MetaspaceOOM {
    public static void main(String[] args) {
        while (true) {
            // 使用CGLib动态生成类
            Enhancer enhancer = new Enhancer();
            enhancer.setSuperclass(Object.class);
            enhancer.setCallback((MethodInterceptor) (obj, method, args1, proxy)
                -> proxy.invokeSuper(obj, args1));
            enhancer.create();  // 不断创建新类
        }
    }
}
```

## 九、总结对比表

| 对比项           | 永久代 (PermGen)   | 元空间 (Metaspace)       |
| ---------------- | ------------------ | ------------------------ |
| **存储位置**     | JVM 堆内存         | 本地内存                 |
| **默认大小**     | 固定（需手动设置） | 无上限（受系统内存限制） |
| **OOM 风险**     | 较高               | 较低                     |
| **GC 效率**      | Full GC 回收       | 更灵活                   |
| **字符串常量池** | 在永久代中         | 移至堆中                 |
| **调优参数**     | PermSize           | MetaspaceSize            |

> 💡 **为什么用元空间替代永久代？**
>
> 1. 永久代大小难以确定，容易 OOM
> 2. 永久代 GC 效率低
> 3. 方便 HotSpot 与 JRockit 合并
