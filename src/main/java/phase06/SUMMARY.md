```
┌────────────────────────────────────────────────────┐
│               运行时数据区                          │
├─────────────────────┬──────────────────────────────┤
│   线程私有区域       │       线程共享区域           │

│  程序计数器          │         堆 (Heap)           │
│  虚拟机栈 (Java栈)   │       方法区 (元空间)       │
│  本地方法栈          │                             │
└─────────────────────┴──────────────────────────────┘
```

| 区域       | 作用             | 异常               |
| ---------- | ---------------- | ------------------ |
| 程序计数器 | 当前线程执行位置 | 无                 |
| 虚拟机栈   | 方法调用栈帧     | StackOverflowError |
| 堆         | 对象实例、数组   | OutOfMemoryError   |
| 方法区     | 类信息、常量池   | OutOfMemoryError   |

**GC 算法对比**:
| 算法 | 优点 | 缺点 | 应用 |
|------|------|------|------|
| 标记-清除 | 简单 | 内存碎片 | 早期实现 |
| 复制算法 | 无碎片、快速 | 空间减半 | 新生代 |
| 标记-整理 | 无碎片 | 移动成本 | 老年代 |

**主流收集器**:
| 收集器 | 特点 | 适用场景 |
|--------|------|----------|
| G1 | 可预测停顿 | 通用 (默认) |
| ZGC | 超低延迟 (<10ms) | 大内存、低延迟 |
| Shenandoah | 低延迟 | 类似 ZGC |

**Java 引用类型**:

```java

// 软引用 - 内存不足时回收

**类加载过程**:

```
加载 → 验证 → 准备 → 解析 → 初始化 → 使用 → 卸载
```

**类加载器层次**:

```
Bootstrap ClassLoader (启动) ← null
         ↑
Platform ClassLoader (平台/扩展)
         ↑
Application ClassLoader (应用)
         ↑
Custom ClassLoader (自定义)
```

**双亲委派**:

- 向上委派：子加载器先委派给父加载器
- 向下加载：父加载器加载失败时子加载器尝试
- 优点：保证类唯一性、防止核心类被篡改

```

## 💡 重点心得

### JVM 问题排查技巧

```
问题类型           排查工具           常见原因
─────────────────────────────────────────────────
CPU 高            jstack + top      死循环、频繁GC
内存溢出          jmap + MAT        大对象、内存泄漏
线程死锁          jstack            锁顺序问题
频繁 Full GC      jstat + GC日志    老年代太小、大对象
```

### 常见 OOM 及解决

| 异常            | 原因        | 解决方案              |
| --------------- | ----------- | --------------------- |
| Java heap space | 堆内存不足  | 增大 -Xmx 或优化代码  |
| Metaspace       | 元空间不足  | 增大 MaxMetaspaceSize |
| GC overhead     | GC 耗时过长 | 优化代码或增大内存    |
| StackOverflow   | 栈深度超限  | 检查递归或增大 -Xss   |

### 最佳实践

1. **内存设置**

   - -Xms 和 -Xmx 设置相同，避免动态调整
   - 堆大小通常为可用内存的 60-80%

2. **GC 选择**

   - JDK11+: 优先考虑 G1 或 ZGC
   - 低延迟场景: ZGC (<10ms)
   - 吞吐量优先: Parallel GC

3. **监控告警**
   - 堆使用率 > 80% 预警
   - GC 时间占比 > 5% 关注
   - Full GC 频率异常监控
