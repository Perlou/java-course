## 💾 内存区域

### 堆内存结构

```
┌─────────────────────────────────────────────────────────────────────┐
│                          堆内存结构                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                      年轻代 (Young Gen)                       │   │
│  │  ┌───────────────────────┬──────────┬──────────┐            │   │
│  │  │        Eden           │    S0    │    S1    │            │   │
│  │  │   (新对象分配区)      │ Survivor │ Survivor │            │   │
│  │  │        80%            │   10%    │   10%    │            │   │
│  │  └───────────────────────┴──────────┴──────────┘            │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                              ↓ 对象晋升                             │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                      老年代 (Old Gen)                         │   │
│  │                   (长期存活的对象)                            │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  对象分配流程:                                                      │
│  1. 新对象在 Eden 区分配                                           │
│  2. Eden 满时触发 Minor GC                                         │
│  3. 存活对象移到 Survivor 区                                       │
│  4. 多次存活后晋升到老年代                                         │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 栈与堆的区别

| 特性     | 栈 (Stack)         | 堆 (Heap)    |
| -------- | ------------------ | ------------ |
| 存储内容 | 局部变量、方法调用 | 对象实例     |
| 线程     | 线程私有           | 线程共享     |
| 生命周期 | 方法结束自动释放   | GC 回收      |
| 大小     | 较小 (MB 级)       | 较大 (GB 级) |
| 速度     | 快                 | 慢           |

## 🔄 类加载

### 类加载过程

```
┌─────────────────────────────────────────────────────────────────────┐
│                        类加载过程                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  加载 (Loading)                                                     │
│  ├── 通过类全限定名获取 .class 文件                                │
│  └── 生成 Class 对象                                               │
│           ↓                                                         │
│  链接 (Linking)                                                     │
│  ├── 验证: 确保字节码格式正确                                      │
│  ├── 准备: 为静态变量分配内存，设置默认值                          │
│  └── 解析: 将符号引用转为直接引用                                  │
│           ↓                                                         │
│  初始化 (Initialization)                                            │
│  └── 执行静态代码块和静态变量赋值                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 🗑️ 垃圾回收

### 判断对象存活

```
可达性分析算法:
从 GC Roots 出发，遍历引用链
不可达的对象被判定为可回收

GC Roots 包括:
• 虚拟机栈中引用的对象
• 静态变量引用的对象
• 常量引用的对象
• 本地方法栈中 JNI 引用的对象
```

### GC 收集器对比

| 收集器   | 特点                 | 适用场景       |
| -------- | -------------------- | -------------- |
| Serial   | 单线程               | 客户端应用     |
| Parallel | 多线程，吞吐量优先   | 批处理任务     |
| CMS      | 并发标记清除         | 低延迟应用     |
| G1       | 分区收集，可预测停顿 | 大堆、平衡场景 |
| ZGC      | 低延迟 (<10ms)       | 超大堆         |

### 常用 JVM 参数

```bash
# 堆内存
-Xms4g                    # 初始堆大小
-Xmx4g                    # 最大堆大小
-Xmn2g                    # 年轻代大小

# GC 收集器
-XX:+UseG1GC              # 使用 G1
-XX:MaxGCPauseMillis=200  # 目标停顿时间

# GC 日志
-Xlog:gc*:file=gc.log

# 调试
-XX:+HeapDumpOnOutOfMemoryError
-XX:HeapDumpPath=/path/to/dump
```

## 🔧 JVM 调优

### 常见问题

```
OOM (OutOfMemoryError):
• Java heap space - 堆内存不足
• Metaspace - 元空间不足
• unable to create native thread - 线程数过多

性能问题:
• 频繁 GC - 堆内存过小或对象创建过多
• Full GC - 老年代满或晋升失败
• GC 停顿长 - 堆太大或收集器不合适
```

### 监控命令

```bash
jps          # 查看 Java 进程
jstat -gc <pid> 1000   # GC 统计
jmap -heap <pid>       # 堆内存信息
jmap -histo <pid>      # 对象统计
jstack <pid>           # 线程堆栈
jcmd <pid> GC.heap_dump /path/dump.hprof  # 堆转储
```
