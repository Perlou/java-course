### 核心概念

| 概念            | 说明                                             |
| --------------- | ------------------------------------------------ |
| **Driver**      | 运行 main() 函数的进程，创建 SparkContext        |
| **Executor**    | Worker 节点上的进程，执行 Task                   |
| **Application** | 一个 Spark 应用，包含一个 Driver 和多个 Executor |
| **Job**         | 由 Action 触发的计算任务                         |
| **Stage**       | Job 按 Shuffle 边界划分的阶段                    |
| **Task**        | Stage 中每个分区的计算单元                       |

## 📊 RDD 核心概念

### RDD 五大特性

```
RDD (Resilient Distributed Dataset) 弹性分布式数据集
═══════════════════════════════════════════════════════════════════

1. 分区列表 (Partitions)
   - 数据被切分为多个分区并行处理
   - 分区数决定任务并行度

2. 计算函数 (Compute)
   - 每个分区都有一个计算函数
   - 定义如何计算该分区的数据

3. 依赖关系 (Dependencies)
   - 记录 RDD 之间的依赖关系
   - 窄依赖 vs 宽依赖

4. 分区器 (Partitioner)
   - 可选，用于 Key-Value RDD
   - HashPartitioner / RangePartitioner

5. 首选位置 (Preferred Locations)
   - 数据本地性优化
   - 尽量在数据所在节点计算
```

### 宽依赖 vs 窄依赖

```
窄依赖 (Narrow Dependency)
═══════════════════════════════════════════════════════════
父 RDD 的每个分区最多被一个子 RDD 分区使用

    父 RDD     子 RDD
   ┌───┐      ┌───┐
   │ P1│ ───→ │ P1│   map, filter, union
   ├───┤      ├───┤   不需要 Shuffle
   │ P2│ ───→ │ P2│   可以流水线执行
   └───┘      └───┘


宽依赖 (Wide Dependency)
═══════════════════════════════════════════════════════════
父 RDD 的每个分区被多个子 RDD 分区使用

    父 RDD        子 RDD
   ┌───┐   ╲    ┌───┐
   │ P1│ ──╲───→│ P1│   groupByKey, reduceByKey
   ├───┤    ╲╱  ├───┤   需要 Shuffle
   │ P2│ ──╱──→ │ P2│   Stage 划分边界
   └───┘   ╱    └───┘
```

### 常用算子

```
转换算子 (Transformation) - 惰性求值
═══════════════════════════════════════════════════════════
map(func)         对每个元素应用函数
filter(func)      过滤元素
flatMap(func)     扁平化映射
groupByKey()      按 Key 分组
reduceByKey(func) 按 Key 聚合
join(other)       连接两个 RDD
distinct()        去重
union(other)      合并

行动算子 (Action) - 触发计算
═══════════════════════════════════════════════════════════
collect()         收集所有数据到 Driver
count()           计数
first()           返回第一个元素
take(n)           取前 n 个元素
reduce(func)      聚合所有元素
saveAsTextFile()  保存到文件
foreach(func)     对每个元素执行操作
```

### Structured Streaming

```
Structured Streaming (Spark 2.0+)
═══════════════════════════════════════════════════════════════════

基于 DataFrame/Dataset API 的流处理
• 将流数据视为无界表 (Unbounded Table)
• 增量查询，结果持续更新
• 支持 Exactly-Once 语义

输出模式：
┌──────────┬──────────────────────────────────────────────────┐
│ Complete │ 每次输出完整结果表（适合聚合查询）               │
├──────────┼──────────────────────────────────────────────────┤
│ Append   │ 只输出新增行（适合简单查询）                     │
├──────────┼──────────────────────────────────────────────────┤
│ Update   │ 只输出更新的行（适合增量更新）                   │
└──────────┴──────────────────────────────────────────────────┘
```

### 数据倾斜处理

```
数据倾斜问题
═══════════════════════════════════════════════════════════════════
少数 Task 处理大量数据，导致整体作业时间过长

解决方案：

1. 增加并行度
   spark.sql.shuffle.partitions = 200 → 更多分区

2. 两阶段聚合（加盐打散）
   原始 key: "hot_key"
   加盐: "hot_key_0", "hot_key_1", ... "hot_key_9"
   第一次聚合后去盐，再全局聚合

3. Broadcast Join
   小表广播到所有节点，避免 Shuffle
   spark.sql.autoBroadcastJoinThreshold = 10MB

4. 自定义分区器
   根据数据分布自定义分区逻辑
```

### 常用配置参数

```bash
# 执行资源
--executor-memory 4G
--executor-cores 4
--num-executors 10

# Shuffle 优化
spark.sql.shuffle.partitions=200
spark.shuffle.file.buffer=64k
spark.reducer.maxSizeInFlight=96m

# 序列化
spark.serializer=org.apache.spark.serializer.KryoSerializer

# 动态资源分配
spark.dynamicAllocation.enabled=true
spark.dynamicAllocation.minExecutors=2
spark.dynamicAllocation.maxExecutors=100
```

## 📝 Spark vs MapReduce

| 特性           | MapReduce          | Spark                          |
| -------------- | ------------------ | ------------------------------ |
| **计算模型**   | 磁盘批处理         | 内存计算                       |
| **速度**       | 慢（中间结果落盘） | 快 10-100 倍                   |
| **API**        | 低级 Map/Reduce    | 丰富的 RDD/DataFrame API       |
| **迭代计算**   | 低效（每轮落盘）   | 高效（内存缓存）               |
| **流处理**     | 不支持             | Streaming/Structured Streaming |
| **机器学习**   | 需要额外框架       | 内置 MLlib                     |
| **交互式查询** | 不支持             | Spark SQL                      |
| **内存需求**   | 低                 | 高                             |
