| 特性     | 英文     | 含义                     | 典型场景            |
| -------- | -------- | ------------------------ | ------------------- |

## 🏗️ HDFS 架构

### 核心组件

```
┌─────────────────────────────────────────────────────────────────────┐
│                         HDFS 架构                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌─────────────────────────────────────────────────────────────┐  │
│   │                      NameNode (主节点)                       │  │
│   │  ┌─────────────────────────────────────────────────────┐    │  │
│   │  │  • 管理文件系统命名空间（目录树）                      │    │  │
│   │  │  • 维护文件 → Block 的映射                           │    │  │
│   │  │  • 记录 Block → DataNode 的位置                      │    │  │
│   │  │  • 内存存储元数据（fsimage + editlog 持久化）         │    │  │
│   │  └─────────────────────────────────────────────────────┘    │  │
│   └─────────────────────────────────────────────────────────────┘  │
│                              │                                      │
│              ┌───────────────┼───────────────┐                     │
│              ▼               ▼               ▼                     │
│   ┌──────────────┐  ┌──────────────┐  ┌──────────────┐            │
│   │  DataNode 1  │  │  DataNode 2  │  │  DataNode 3  │            │
│   │  ┌────────┐  │  │  ┌────────┐  │  │  ┌────────┐  │            │
│   │  │Block 1 │  │  │  │Block 1 │  │  │  │Block 2 │  │            │
│   │  │Block 2 │  │  │  │Block 3 │  │  │  │Block 3 │  │            │
│   │  └────────┘  │  │  └────────┘  │  │  └────────┘  │            │
│   └──────────────┘  └──────────────┘  └──────────────┘            │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### Block 与副本

```
Block（数据块）
═══════════════════════════════════════════════════════════
• 默认大小：128 MB（Hadoop 2.x+）
• 大文件切分为多个 Block 存储
• 每个 Block 独立存储和复制

副本放置策略（默认3副本）
═══════════════════════════════════════════════════════════
• 第 1 个副本：客户端所在节点（或随机节点）
• 第 2 个副本：不同机架的某个节点
• 第 3 个副本：与第 2 个副本同机架的不同节点

为什么这样设计？
• 机架内传输快 → 提高写入速度
• 跨机架存储 → 容灾，防止整个机架故障
```

## 🔄 MapReduce 编程模型

### 核心思想

```
┌─────────────────────────────────────────────────────────────────┐
│                     MapReduce 执行流程                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  输入        Split 1 ──→ Map 1 ──┐                              │
│  数据        Split 2 ──→ Map 2 ──┼──→ Shuffle ──→ Reduce 1 ──┐  │
│  (HDFS)      Split 3 ──→ Map 3 ──┘       Sort      Reduce 2 ──┼→ 输出
│                                                    Reduce 3 ──┘  │
│                                                                 │
│  ════════════════════════════════════════════════════════════  │
│   Map 阶段          Shuffle 阶段            Reduce 阶段         │
│   (分而治之)          (洗牌排序)              (合并汇总)          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### WordCount 示例

```
输入: "hello world hello"

Map 阶段:
  "hello world hello" → [(hello,1), (world,1), (hello,1)]

Shuffle & Sort:
  [(hello,1), (hello,1)] → (hello, [1,1])
  [(world,1)]            → (world, [1])

Reduce 阶段:
  (hello, [1,1]) → (hello, 2)
  (world, [1])   → (world, 1)

输出: hello 2, world 1
```

### Shuffle 过程详解

```
┌────────────────────────────────────────────────────────────────┐
│                      Shuffle 详细过程                           │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  Map 端 Shuffle:                                               │
│  ┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐       │
│  │  Map    │ → │  环形    │ → │  溢写    │ → │  合并   │       │
│  │  Output │   │  缓冲区  │   │  Spill  │   │ Merge  │       │
│  └─────────┘   │ (100MB) │   │  (排序)  │   │        │       │
│                └─────────┘   └─────────┘   └─────────┘       │
│                    80% 触发溢写                                │
│                                                                │
│  Reduce 端 Shuffle:                                            │
│  ┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐       │
│  │  Copy   │ → │  Merge  │ → │  Sort   │ → │ Reduce  │       │
│  │ (拉取)  │   │  (合并)  │   │  (排序)  │   │        │       │
│  └─────────┘   └─────────┘   └─────────┘   └─────────┘       │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

```
┌─────────────────────────────────────────────────────────────────────┐
│                          YARN 架构                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌─────────────────────────────────────────────────────────────┐  │
│   │                  ResourceManager (全局)                      │  │
│   │  • 资源调度器：分配集群资源                                   │  │
│   │  • 应用管理器：管理所有应用的 ApplicationMaster              │  │
│   └─────────────────────────────────────────────────────────────┘  │
│                              │                                      │
│              ┌───────────────┼───────────────┐                     │
│              ▼               ▼               ▼                     │
│   ┌──────────────┐  ┌──────────────┐  ┌──────────────┐            │
│   │ NodeManager 1│  │ NodeManager 2│  │ NodeManager 3│            │
│   │  ┌────────┐  │  │  ┌────────┐  │  │  ┌────────┐  │            │
│   │  │Container│  │  │  │Container│  │  │  │Container│  │            │
│   │  │   AM   │  │  │  │  Task  │  │  │  │  Task  │  │            │
│   │  └────────┘  │  │  └────────┘  │  │  └────────┘  │            │

│                                                                     │

```

### 调度器对比

| 调度器       | 特点               | 适用场景       |
| ------------ | ------------------ | -------------- |
| **FIFO**     | 先进先出，简单     | 测试环境       |
| **Capacity** | 按队列分配容量     | 多租户共享集群 |
| **Fair**     | 公平分配，动态调整 | 交互式查询     |

## 📁 数据格式对比

| 格式             | 类型 | 压缩 | 分片 | 适用场景                |
| ---------------- | ---- | ---- | ---- | ----------------------- |
| **Text/CSV**     | 行式 | 可选 | ✅   | 简单数据交换            |
| **SequenceFile** | 行式 | ✅   | ✅   | Hadoop 中间结果         |
| **Avro**         | 行式 | ✅   | ✅   | 数据序列化，Schema 演进 |
| **Parquet**      | 列式 | ✅   | ✅   | OLAP 分析查询           |
| **ORC**          | 列式 | ✅   | ✅   | Hive 优化存储           |

```
列式存储 vs 行式存储
═══════════════════════════════════════════════════════════

行式存储（适合 OLTP）:
┌───────┬───────┬───────┐
│ id=1  │ name  │ age   │  → 一行数据连续存储
├───────┼───────┼───────┤
│ id=2  │ name  │ age   │
└───────┴───────┴───────┘

列式存储（适合 OLAP）:
┌───────────────────────┐
│ id: 1, 2, 3, 4, 5 ... │  → 同列数据连续存储
├───────────────────────┤
│ name: a, b, c, d ...  │  → 压缩率高，只读需要的列
├───────────────────────┤
│ age: 20, 25, 30 ...   │
└───────────────────────┘
```

## 🛠️ 常用命令

### HDFS 命令

```bash
# 文件操作
hdfs dfs -ls /                    # 列目录
hdfs dfs -mkdir /user/data        # 创建目录
hdfs dfs -put local.txt /user/    # 上传文件
hdfs dfs -get /user/file.txt ./   # 下载文件
hdfs dfs -cat /user/file.txt      # 查看内容
hdfs dfs -rm -r /user/data        # 删除目录

# 管理命令
hdfs dfsadmin -report             # 集群状态
hdfs fsck / -files -blocks        # 健康检查
```

### MapReduce 命令

```bash
# 提交作业
hadoop jar wordcount.jar WordCount /input /output

# 查看作业状态
mapred job -list
mapred job -kill job_xxx
```

4. **熟悉命令行** - 日常运维必备技能
5. **关注生态演进** - Spark/Flink 正在替代 MapReduce
