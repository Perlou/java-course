## 1. 产品概述

### 1.1 产品定位

TurboX Exchange 是一个面向数字资产的交易平台，提供现货交易功能，支持多种交易对的撮合、资金管理和行情展示。

### 1.2 目标用户

| 用户类型   | 描述                   |
| ---------- | ---------------------- |
| 普通交易者 | 进行买卖操作的终端用户 |
| 做市商     | 提供流动性的专业交易者 |
| 平台管理员 | 管理平台运营的后台人员 |

### 1.3 核心价值

- **高效**：毫秒级撮合，实时行情
- **安全**：资金安全，风控完善
- **稳定**：高可用架构，99.9% SLA

#### 2.3.4 精度与数量规范

所有金额、价格、数量字段**必须全程使用 `BigDecimal`**，严禁 `double`/`float`。

**交易对级别配置表**（示例）

| 参数 | 说明 | BTC/USDT | ETH/USDT |
| ---- | ---- | -------- | -------- |
| pricePrecision | 价格小数位 | 2 | 2 |
| quantityPrecision | 数量小数位 | 6 | 4 |
| minQuantity | 最小下单量 | 0.000001 | 0.0001 |
| maxQuantity | 最大下单量 | 100 | 5,000 |
| minNotional | 最小名义金额 | 10 USDT | 10 USDT |
| tickSize | 价格最小变动 | 0.01 | 0.01 |
| stepSize | 数量最小变动 | 0.000001 | 0.0001 |

**舍入规则**

- 统一使用 `RoundingMode.DOWN`（截断，不四舍五入）
- 手续费计算结果向上取整至最小精度（对平台有利）
- 下单校验：数量和价格必须是 stepSize / tickSize 的整数倍，否则拒绝

### 2.8 手续费体系

#### 2.8.1 Maker/Taker 基础费率

| 角色  | 基础费率 | 判定规则 |
| ----- | -------- | -------- |
| Maker | 0.1%     | 订单进入订单簿后被动成交（挂单方） |
| Taker | 0.2%     | 订单主动吃掉已有订单（吃单方） |

> **判定规则**：限价单下单后未立即成交、进入订单簿的为 Maker；市价单和立即成交的限价单为 Taker。

#### 2.8.2 阶梯费率表

按用户近 30 天交易量（USDT 等值）分级：

| 等级 | 30 天交易量 | Maker 费率 | Taker 费率 |
| ---- | ----------- | ---------- | ---------- |
| VIP0 | < 100,000   | 0.10%      | 0.20%      |
| VIP1 | ≥ 100,000   | 0.08%      | 0.18%      |
| VIP2 | ≥ 1,000,000 | 0.06%      | 0.15%      |
| VIP3 | ≥ 10,000,000| 0.04%      | 0.10%      |

#### 2.8.3 扣费币种规则

| 订单方向 | 扣费币种 | 示例（BTC/USDT） |
| -------- | -------- | ----------------- |
| 买单     | base 币种 | 买入 1 BTC，手续费 0.002 BTC |
| 卖单     | quote 币种 | 卖出 1 BTC 得 50,000 USDT，手续费 100 USDT |

**手续费计算公式**

```
买单手续费 = 成交数量 × 费率
卖单手续费 = 成交数量 × 成交价格 × 费率
```

### 2.9 错误场景与边界条件

#### 2.9.1 下单相关

| 场景 | 处理规则 | 错误码 |
| ---- | -------- | ------ |
| 余额不足 | 拒绝下单 | INSUFFICIENT_BALANCE |
| 价格偏离市价 > 10% | 拒绝下单 | PRICE_DEVIATION |
| 精度超出交易对配置 | 拒绝下单 | PRECISION_EXCEEDED |
| 数量低于 minQuantity | 拒绝下单 | QUANTITY_TOO_SMALL |
| 名义金额低于 minNotional | 拒绝下单 | NOTIONAL_TOO_SMALL |
| 交易对暂停交易 | 拒绝下单 | SYMBOL_SUSPENDED |
| 用户被冻结 | 拒绝下单 | ACCOUNT_FROZEN |

#### 2.9.2 撮合相关

| 场景 | 处理规则 |
| ---- | -------- |
| 自成交 | 取消后到达的订单（passive reject），先进入订单簿的保留 |
| IOC 订单未完全成交 | 立即撤销未成交部分 |
| FOK 订单无法全部成交 | 整笔拒绝，不进入订单簿 |
| 市价买单无对手盘 | 拒绝下单 |
| 市价卖单无对手盘 | 拒绝下单 |

#### 2.9.3 资金相关

| 场景 | 处理规则 |
| ---- | -------- |
| 结算时账户已冻结 | 仍然完成结算（资金安全优先） |
| 并发扣款导致余额不足 | `available >= 0` CHECK 约束 + 乐观锁重试 |
| Kafka 消息发送失败 | Outbox Pattern 保证最终投递 |
| 重复结算请求 | 幂等检查（biz_id UNIQUE KEY），跳过已处理 |
